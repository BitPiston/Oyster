Oyster Web Server

A simple, quick and dirty, web server that can (and should only!) be used to deploy quick development environments.

Command Line Arguments

-site (optional) Specifies a particular site ID to use -env (optional) Specifies a particular configuration environment to use -host (optional) Specifies an ip address or host to bind to, defaults to to 127.0.0.1 -port (optional) Specifies a port to bind to, defaults to 80 Host and port can alternatively be passed as a single string, without a name. For example: perl script/server.pl 192.168.1.100:82 perl script/server.pl 192.168.1.100 perl script/server.pl 82

Each time server.pl runs it saves its configuration and will automatically re-use its last configuration if host and port are not specified.
